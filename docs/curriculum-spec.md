## 0. 全体方針

### 0.1 レッスンの基本構造

全レッスンは「**解説 → 実行例 → 演習問題(自動判定)**」の3部構成。1レッスン10〜15分で完了できる分量に抑える(解説はスクロール3〜4画面以内、演習は1問)。

- **解説**: 専門用語には必ず平易な補足を付ける(例: 「変数(データに名前を付けて覚えておく箱)」)。初出の用語は太字。コード例は5行以内を基本とする。
- **実行例**: 解説内容に対応した、そのまま実行できる完結したコード。学習者はワンクリックでエディタに読み込み、実行・改変できる。
- **演習**: 問題文、初期コード(雛形)、2段階ヒント、模範解答、判定テストコードを持つ。

### 0.2 判定方式の設計原則

判定は Pyodide(Web Worker)上で次のパイプラインで行う:

1. 新しいグローバル名前空間 `ns` を用意する(レッスンごとに毎回リセット。前回実行の変数は残らない)
2. `sys.stdout` を `io.StringIO` に差し替えて **ユーザーコードを `exec(user_code, ns)` で実行**
3. ユーザーコードが例外で終了した場合 → その時点で**不合格**とし、エラー内容(トレースバック末尾)を表示
4. キャプチャした標準出力の全文を文字列 `__stdout__` として `ns` に注入
5. **判定テストコードを `exec(test_code, ns)` で実行**(同じ名前空間なので、ユーザーの変数・関数・`__stdout__` を参照できる)
6. テストコードが例外なく完走 → **合格**。`AssertionError` → **不合格**とし、その日本語メッセージを学習者へのフィードバックとして表示。それ以外の例外 → 不合格とし「判定中にエラーが発生しました」+ 詳細を表示

**検証対象の使い分け**(何をassertするか):

| 検証対象 | 使う場面 | 方法 |
|---|---|---|
| 変数の値 | 第2〜7章の中心。計算結果・データ操作結果 | `assert total == 200, "..."` |
| 標準出力 | 第1章の中心。print系レッスン、ループの出力 | `__stdout__` を行分割して比較(0.3節) |
| 関数の戻り値 | **第8章以降の中心**。複数の入力で呼び出して検証 | `assert add(2, 3) == 5, "..."` |
| 関数のprint出力 | 第8章の一部 | テスト内でstdoutを差し替えて呼び出す(0.3節) |
| 入力データの不変性 | リスト・辞書系。雛形の入力変数を書き換え/削除していないか | `assert numbers == [30, 10, ...], "..."` |

**テストコードの記述規約**(content-writerはこれに厳密に従う):

- assertメッセージは必ず日本語で、「何が期待と違うか」を伝えるが**答えそのものは書かない**(例: 「変数 total の値が正しくありません。合計の計算を見直しましょう」)
- テストコード内で定義する補助変数・補助関数はすべてアンダースコア始まり(`_lines`、`_buf` 等)にして、ユーザー変数を上書きしない
- 変数・関数の存在チェックを値チェックより先に行う: `assert "total" in globals(), "変数 total が見つかりません。total という名前で作りましたか?"`
- float比較は `abs(x - 期待値) < 1e-6` を使う
- `import` はテストコード内でのみ `io`, `sys` を許可(関数のprint出力キャプチャ用)。それ以外のimportはテストコードでも使わない
- 関数はできる限り**3ケース以上**(典型・境界・意地悪)で呼び出す

**合否品質の必須条件**(exercise-validatorが全演習で検証する):

1. 模範解答は必ず合格する
2. 初期コード(雛形)を無変更で実行すると必ず不合格になる(雛形に `answer = None` 等のプレースホルダを置く、または未完成のままにすることで保証)
3. そのレッスンで指摘する典型的な誤答(仕様に明記)が不合格になる

### 0.3 print出力を検証する演習の判定方法

`__stdout__` はユーザーコードの標準出力全文(改行含む)。比較は**行単位・末尾空白無視**を標準とする。各テストコードの冒頭に次の定型を置く:

```python
_lines = [_l.rstrip() for _l in __stdout__.rstrip("\n").split("\n")] if __stdout__.strip() else []
```

- 完全一致判定: `assert _lines == ["こんにちは"], "..."`(行数も内容もチェックされる)
- 行末の空白・最終行後の改行の有無は不問(rstripするため)
- 部分一致(「出力に◯◯が含まれる」)は、変数検証が主でprintは確認目的、という演習でのみ使う: `assert "200" in __stdout__, "total を print で表示してください"`

**関数のprint出力**を検証する場合はテストコード内で再キャプチャする(第8章で使用):

```python
import io, sys
_buf = io.StringIO()
_old = sys.stdout
sys.stdout = _buf
try:
    greet("たろう")
finally:
    sys.stdout = _old
assert _buf.getvalue().rstrip() == "こんにちは、たろうさん!", "..."
```

なお判定テストコード自身のprint出力は破棄され、学習者には見えない(worker側でテスト実行中もstdoutを捨てバッファに向ける)。

### 0.4 `input()` を使わない方針と代替パターン

Pyodide環境では `input()` は使えないため、**カリキュラム全体で一切使わない**。「ユーザーからの入力」は次のパターンで代替する:

- **雛形の冒頭で変数に代入して与える**。コメントで「この値が入力だと思ってください」と明示する:
  ```python
  temperature = 32  # ← この値をいろいろ変えて試してみよう(判定は 32 のままで行います)
  ```
- 問題文には「変数 ◯◯ に入っている値を使って」と書き、値を直書きさせない誘導をする
- 第8章で関数を学んだ後は、**入力=引数、出力=戻り値**の形式に移行する。テストは複数の引数で呼び出すため、値の直書き(ハードコード)は自然に不合格になる

**設計上の割り切り(重要)**: 第7章までは入力が固定値のため、結果だけを直書きした解答(例: `total = 5050` と書くだけ)も技術的には合格する。完全防止には関数化が必要だが未習であるため、ここは**教育的誘導(問題文・ヒント・模範解答)で正しい書き方に導く**方針とし、機械的防止は第8章以降で行う。この割り切りは全関係者の共通認識とする。

### 0.5 実行環境の制約(演習設計に効くもの)

- `input()` 禁止(0.4節)。ファイルI/O・ネットワーク・`import`(標準ライブラリ含む)は**演習コード・模範解答では使わない**。テストコードのみ `io`/`sys` 可
- 無限ループは10秒でWorkerが強制停止される。第5章while回で「固まっても壊れない、10秒で止まる」ことを解説に明記する
- 演習の計算量はごく小さく保つ(ループは最大でも数百回)

### 0.6 レッスンID・依存関係の表記

- レッスンID: `chNN-MM`(章番号2桁-レッスン番号2桁)。例 `ch01-01`
- 各レッスンの「前提」には直接依存するレッスンIDのみ書く(同章の直前レッスンは暗黙の前提とし省略可だが、章をまたぐ依存は必ず明記)
- 未習事項を演習に使わないこと。章末の「10.5 依存関係の点検結果」に前倒し導入した項目を列挙する

### ch01-01 はじめてのプログラム

- **学習目標**
  1. `print()` を使って文字列を画面に表示できる
  2. 「コードを書いて実行ボタンを押すと結果が出る」という流れを体験する
- **前提知識**: なし(最初のレッスン)
- **解説の骨子**
  1. **プログラムってなに?** — コンピュータへの指示書。上から1行ずつ実行される。このサイトではブラウザの中でPythonがそのまま動くこと、何を書いても壊れないことを伝えて安心させる
  2. **print で表示してみよう** — `print("こんにちは")` を提示。「print(表示したいもの)」という形。文字は `"`(ダブルクォート)で囲む、囲んだものを**文字列**と呼ぶ(用語補足: 文字の並びのこと)
  3. **2行書けば2回表示される** — print2行のコード例で「上から順に実行」を見せる
- **実行例サンプルコード**
  ```python
  print("こんにちは")
  print("Pythonをはじめよう")
  ```
- **演習問題**
  - **問題文の要旨**: 画面に `こんにちは、Python!` と1行だけ表示するプログラムを書いてください(`!` は半角)
  - **初期コード(雛形)**:
    ```python
    # ここに print を使ったコードを書いてください
    ```
  - **判定基準(テストコード実コード)**:
    ```python
    _lines = [_l.rstrip() for _l in __stdout__.rstrip("\n").split("\n")] if __stdout__.strip() else []
    assert len(_lines) > 0, "何も表示されていません。print() を使って表示してみましょう"
    assert len(_lines) == 1, "表示が1行ではありません。print は1つだけ書きましょう"
    assert _lines[0] == "こんにちは、Python!", "表示された文字が違います。「こんにちは、Python!」と正確に表示しましょう(記号も含めて)"
    ```
  - **不合格になるべき誤答**: 空回答(何も出力なし→1つ目のassert)/ `こんにちは、Python`(!忘れ→3つ目)/ 2行に分けて出力(2つ目)
  - **ヒント1(考え方)**: 表示には `print()` を使います。表示したい文字は `"` で囲んでカッコの中に入れます
  - **ヒント2(ほぼ答え)**: `print("こんにちは、Python!")` — カッコとクォートの対応に注意
  - **模範解答**: `print("こんにちは、Python!")` 1行

### ch01-02 文字と数を表示する

- **学習目標**
  1. 数値はクォートなしで print できることを理解する
  2. `print(a, b)` のようにカンマで複数の値を並べて表示できる
- **前提知識**: ch01-01
- **解説の骨子**
  1. **数はクォートなしで書ける** — `print(42)` と `print("42")` はどちらも 42 と表示されるが、意味が違う(数と文字)。詳しくは第2章でやると予告
  2. **カンマで並べて表示** — `print("答えは", 42)` → `答えは 42`。カンマで区切ると自動で半角スペースが入る
  3. **クォートの付け忘れ・付けすぎに注意** — `print(こんにちは)` はエラーになる(エラーの読み方は ch01-04 で)
- **実行例サンプルコード**
  ```python
  print(42)
  print("答えは", 42)
  print(1, 2, 3)
  ```
- **演習問題**
  - **問題文の要旨**: 次の3行を順番に表示する。1行目 `好きな数`、2行目に数値の `42`(クォートなしで書くこと)、3行目 `答えは 42`(カンマ区切りで「答えは」と 42 の2つを並べる)
  - **初期コード(雛形)**:
    ```python
    # 3行表示するプログラムを書いてください
    # 1行目: 好きな数
    # 2行目: 42 (数値として)
    # 3行目: 答えは 42 (カンマで2つの値を並べる)
    ```
  - **判定基準(テストコード実コード)**:
    ```python
    _lines = [_l.rstrip() for _l in __stdout__.rstrip("\n").split("\n")] if __stdout__.strip() else []
    assert len(_lines) == 3, "表示が3行になっていません。print を3つ書きましょう"
    assert _lines[0] == "好きな数", "1行目が「好きな数」になっていません"
    assert _lines[1] == "42", "2行目が「42」になっていません"
    assert _lines[2] == "答えは 42", "3行目が「答えは 42」になっていません。print(\"答えは\", 42) のようにカンマで並べます"
    ```
    ※ stdoutでは `42` と `"42"` は区別できないため、「クォートなしで」は教育的指示にとどめ機械判定しない(仕様として明記)
  - **不合格になるべき誤答**: 行の過不足/`答えは42`(スペースなし: カンマを使わず連結した場合)
  - **ヒント1**: print は書いた順に1行ずつ表示します。カンマで区切ると間に半角スペースが自動で入ります
  - **ヒント2**: `print("好きな数")`、`print(42)`、`print("答えは", 42)` の3行です
  - **模範解答**: 上記3行

### ch01-03 コメント — メモを書いても動く

- **学習目標**
  1. `#` から行末まではコメント(実行されないメモ)だと理解する
  2. コメントアウトで「実行したくない行」を一時的に無効化できる
- **前提知識**: ch01-01
- **解説の骨子**
  1. **コメント=プログラム内のメモ** — `#` の後ろは実行されない。日本語でOK。自分や他人のための説明を書く文化を紹介
  2. **行の後ろにも書ける** — `print("hi")  # あいさつ` の例
  3. **コメントアウトという技** — エラーになる行や試したくない行の先頭に `#` を付けて無効化する。デバッグの第一歩として紹介
- **実行例サンプルコード**
  ```python
  # このプログラムはあいさつをします
  print("おはよう")  # 朝のあいさつ
  # print("この行は実行されません")
  ```
- **演習問題**
  - **問題文の要旨**: 雛形の3行のうち真ん中の行(コードではないメモ)が原因でエラーになる。その行をコメントアウトして、`おはよう` と `おやすみ` の2行が表示されるように直す
  - **初期コード(雛形)**:
    ```python
    print("おはよう")
    ここは自分用のメモです
    print("おやすみ")
    ```
    (雛形のまま実行すると2行目で NameError となり不合格 → 条件0.2-2を満たす)
  - **判定基準(テストコード実コード)**:
    ```python
    _lines = [_l.rstrip() for _l in __stdout__.rstrip("\n").split("\n")] if __stdout__.strip() else []
    assert len(_lines) == 2, "表示が2行になっていません。メモの行だけを無効にして、print の2行は残しましょう"
    assert _lines[0] == "おはよう", "1行目が「おはよう」になっていません"
    assert _lines[1] == "おやすみ", "2行目が「おやすみ」になっていません"
    ```
    ※ メモ行を削除した解答も合格とする(コメントアウトも削除も「実行させない」正しい対処のため)。ただし模範解答はコメントアウトで示す
  - **ヒント1**: 実行したくない行は、行の先頭に `#` を付けると「コメント」になり、Pythonから見えなくなります
  - **ヒント2**: 2行目を `# ここは自分用のメモです` に書き換えます
  - **模範解答**: 2行目の先頭に `# ` を付けた3行

### ch01-04 エラーと友達になる

- **学習目標**
  1. エラーメッセージの最後の行に「エラーの種類: 説明」が書かれていることを知る
  2. SyntaxError(書き方の間違い)と NameError(名前の間違い)を自力で直せる
- **前提知識**: ch01-01, ch01-03
- **解説の骨子**
  1. **エラーは失敗ではなくヒント** — エラーが出てもコンピュータは壊れない。Pythonが「ここが分からないよ」と教えてくれている
  2. **エラーメッセージの読み方** — 最後の行を見る。`SyntaxError: ...` は文法(書き方)の間違い: カッコやクォートの閉じ忘れが定番。行番号も表示される
  3. **NameError** — `NameError: name 'prin' is not defined` を例に、スペルミス・クォート忘れで起きることを説明
  4. **直し方の手順** — ①最後の行でエラーの種類を見る ②示された行番号の周辺を見る ③カッコ・クォート・スペルを確認
- **実行例サンプルコード**(わざとエラーを見るコード。実行してエラー表示を観察させる)
  ```python
  print("エラーを見てみよう"
  ```
  (解説内で、これを実行すると SyntaxError になること、閉じカッコを足すと直ることを示す)
- **演習問題**
  - **問題文の要旨**: 雛形には間違いが2つある(1行目: 閉じカッコ忘れ→SyntaxError、2行目: printのスペルミス→NameError)。エラーメッセージを読みながら2つとも直し、`エラーを` と `直せたよ` の2行を表示させる
  - **初期コード(雛形)**:
    ```python
    print("エラーを"
    prlnt("直せたよ")
    ```
  - **判定基準(テストコード実コード)**:
    ```python
    _lines = [_l.rstrip() for _l in __stdout__.rstrip("\n").split("\n")] if __stdout__.strip() else []
    assert len(_lines) == 2, "表示が2行になっていません。2つの print が両方動くように直しましょう"
    assert _lines[0] == "エラーを", "1行目が「エラーを」になっていません"
    assert _lines[1] == "直せたよ", "2行目が「直せたよ」になっていません"
    ```
  - **不合格になるべき誤答**: 片方だけ直した状態(実行時エラーで不合格)/エラー行を消しただけ(行数不足で不合格)
  - **ヒント1**: エラーメッセージの最後の行にエラーの種類が出ます。1つ目は「閉じカッコ」、2つ目は「printのつづり」を確認しましょう。エラーは1度に1つずつしか出ないので、直しては実行を繰り返します
  - **ヒント2**: 1行目は `print("エラーを")`、2行目は `print("直せたよ")` です
  - **模範解答**: 修正済みの2行

### ch02-01 変数 — データに名前を付ける

- **学習目標**
  1. `名前 = 値` で変数を作り、printで中身を表示できる
  2. 変数は後から別の値で上書きできることを理解する
- **前提知識**: 第1章
- **解説の骨子**
  1. **変数=名前付きの入れ物** — `name = "Python"` の図解(ラベルを貼った箱)。`=` は「等しい」ではなく「右の値を左の名前に入れる」(重要な注意)
  2. **変数を使う** — `print(name)` はクォートなし。`print("name")` との違いを対比で見せる
  3. **上書きできる** — 同じ名前に再代入すると中身が入れ替わるコード例
  4. **名前の付け方** — 半角英小文字と `_`、数字始まりは不可。意味の分かる名前を推奨
- **実行例サンプルコード**
  ```python
  name = "Python"
  print(name)
  name = "パイソン"
  print(name)
  ```
- **演習問題**
  - **問題文の要旨**: 変数 `name` を作って文字列 `"Python"` を入れ、printで表示する
  - **雛形の方針**: コメントで手順を2行示すのみ(コードなし)
  - **判定基準**: ①`assert "name" in globals()`(未定義メッセージ付き) ②`assert name == "Python"`(値) ③`assert isinstance(name, str)`(クォートを忘れて `name = Python` と書くと NameError で実行時不合格になるため、これは保険) ④`assert "Python" in __stdout__`(printしたか)

  - **ヒント1**: `変数名 = 値` で変数が作れます。文字列は `"` で囲みます。表示は `print(変数名)`(クォートなし)
  - **ヒント2**: `name = "Python"` の次の行に `print(name)`
  - **模範解答の方針**: 2行。コメントで「クォートの有無」の違いに触れる

### ch02-03 文字列 — つなげる・繰り返す

### ch03-01 四則演算

- **学習目標**
  1. `+ - * /` で計算できる(`*` と `/` の記号に慣れる)
  2. `/` の結果は常に float になることを知る
- **前提知識**: ch02-02
- **解説の骨子**
  1. **4つの演算子** — 記号は `×→*`、`÷→/`。変数どうしでも計算できる
  2. **/ はいつも小数** — `10 / 2` が `5.0` になる例(int÷intでもfloat)
  3. **優先順位とカッコ** — `2 + 3 * 4` と `(2 + 3) * 4`。迷ったらカッコ
- **実行例サンプルコード**: `a = 7` `b = 3` に対し4演算をprintする5行
- **演習問題**
  - **問題文の要旨**: 雛形の `a = 7`、`b = 3` から、`add`(和)、`sub`(差 a−b)、`mul`(積)、`div`(商 a÷b)の4変数を作る
  - **雛形の方針**: `a`, `b` を与え、4変数名をコメントで指定
  - **判定基準**: ①4変数の存在 ②`add == 10`、`sub == 4`、`mul == 21`、`abs(div - 7/3) < 1e-6` ③`isinstance(div, float)` ④`a == 7 and b == 3`

  - **ヒント1**: 掛け算は `*`、割り算は `/`。`add = a + b` のように変数で計算します
  - **ヒント2**: `add = a + b` / `sub = a - b` / `mul = a * b` / `div = a / b`
  - **模範解答の方針**: 4行+確認用print

### ch03-02 割り算の仲間 — //、%、**

- **学習目標**
  1. `//`(商の整数部分)と `%`(余り)を使い分けられる
  2. `%` が「倍数の判定」に使えることを知る(FizzBuzzへの伏線)
- **前提知識**: ch03-01
- **解説の骨子**
  1. **// と %** — 「17個のあめを5人で分ける」→1人3個(`17 // 5`)で2個余る(`17 % 5`)。生活例で導入
  2. **% の一番の使いみち** — `x % 2 == 0` なら偶数(比較は次レッスンで詳しく、ここでは読み方だけ)。`% 3 == 0` なら3の倍数。第10章FizzBuzzで大活躍と予告
  3. **\*\* べき乗** — `2 ** 5`。`2 * 5` との違いに注意
- **実行例サンプルコード**: あめの分配の3行+`2 ** 5`
- **演習問題**
  - **問題文の要旨**: 雛形の `candies = 17`、`people = 5` から、1人分 `share`(//)、余り `remainder`(%)を作る。さらに `power` に 2の8乗 を `**` で作る
  - **雛形の方針**: 2変数を与え、3変数名を指定
  - **判定基準**: ①存在 ②`share == 3`、`remainder == 2`、`power == 256` ③`candies == 17 and people == 5`
  - **不合格になるべき誤答**: 空回答/`share = candies / people`(3.4で不合格)/`power = 2 * 8`(16)
  - **ヒント1**: 整数の商は `//`、余りは `%`、べき乗は `**` です
  - **ヒント2**: `share = candies // people` / `remainder = candies % people` / `power = 2 ** 8`
  - **模範解答の方針**: 3行+print

### ch03-03 比較とbool — 正しいか間違いか

- **学習目標**
  1. 比較演算子6種の結果が `True` / `False`(bool型)になることを理解する
  2. `=`(代入)と `==`(等しいか)の違いを説明できる
- **前提知識**: ch03-01
- **解説の骨子**
  1. **比較すると答えはTrue/False** — `print(10 < 25)` → `True`。bool型(用語補足: 「はい/いいえ」の2値だけの型)
  2. **6つの比較演算子** — `== != < > <= >=` の表。`<=` の記号の順序(=が後)に注意
  3. **`=` と `==` は別物**(最重要注意)— 代入と質問。間違えると SyntaxError になる例
  4. **boolも変数に入れられる** — `is_less = 10 < 25`。第4章のifで使うと予告
- **実行例サンプルコード**: 比較3種のprintと `is_adult = age >= 18` の例
- **演習問題**
  - **問題文の要旨**: 雛形の `x = 10`、`y = 25` から、`is_equal`(xとyは等しいか)、`is_less`(xはyより小さいか)の2変数を比較演算子で作る
  - **雛形の方針**: 2変数を与え、2変数名と意味を指定
  - **判定基準**: ①存在 ②`is_equal is False`、`is_less is True` ③`isinstance(is_equal, bool)`(`"False"` 等の文字列を弾く) ④入力不変
  - **不合格になるべき誤答**: 空回答/`is_equal = "False"`(③)/不等号の向き逆(②)
  - **ヒント1**: 「等しいか」は `==`(=を2つ)。「より小さい」は `<`。結果をそのまま変数に入れられます
  - **ヒント2**: `is_equal = x == y` / `is_less = x < y`
  - **模範解答の方針**: 2行+print(0.4節の割り切りにより `is_equal = False` 直書きも通るが、ヒントで演算子使用に誘導)

### ch03-04 and・or・not — 条件を組み合わせる

- **学習目標**
  1. `and`(両方)、`or`(どちらか)、`not`(反転)で条件を組み合わせられる
- **前提知識**: ch03-03
- **解説の骨子**
  1. **and: 両方Trueのとき** — 「13歳以上 **かつ** 19歳以下ならティーンエイジャー」を `(13 <= age) and (age <= 19)` で。日常語との対応表
  2. **or: どちらかがTrueのとき** — 「土曜 または 日曜なら休み」の例
  3. **not: ひっくり返す** — `not True` は `False`
  4. **カッコで読みやすく** — 必須ではないがカッコを推奨(初心者向け方針)
- **実行例サンプルコード**: age を変えながら and/or/not の結果をprintする
- **演習問題**
  - **問題文の要旨**: 雛形の `age = 15` から、`is_teen`(13以上 かつ 19以下)と `is_not_teen`(is_teenの反対を `not` で)の2変数を作る
  - **雛形の方針**: `age = 15` を与える(値を変えて試すよう促すが判定は15)
  - **判定基準**: ①存在 ②`is_teen is True`、`is_not_teen is False` ③`isinstance(is_teen, bool)` ④`age == 15`
  - **不合格になるべき誤答**: 空回答/`and` を `or` にする(境界の理解不足はこの値では検出不能のため、ヒントで注意喚起。※値15ではand/orどちらもTrueになるので、**雛形は `age = 25` に変更する**: and→False, or→True で誤りを検出できる。期待値は `is_teen is False`、`is_not_teen is True`)
  - **判定基準(確定版)**: `age = 25` で ②`is_teen is False`、`is_not_teen is True`
  - **ヒント1**: 「AかつB」は `A and B`。「〜でない」は `not 〜`。25歳はティーンエイジャーでしょうか?
  - **ヒント2**: `is_teen = (13 <= age) and (age <= 19)` / `is_not_teen = not is_teen`
  - **模範解答の方針**: 2行+print

## 第4章 条件分岐

**章の狙い**: 「条件によって実行する行を変える」プログラムの流れ制御を身につける。インデントの概念を初導入する重要章。

**判定上の注意**: 本章の演習は入力変数が固定のため出力直書きでも合格し得る(0.4節の割り切り)。分岐の誤りを検出できるよう、**入力値は「誤った分岐だと異なる出力になる値」を選定**している。

### ch04-01 if文 — もし〜なら

- **学習目標**
  1. `if 条件:` とインデントで「条件が成り立つときだけ実行」を書ける
  2. コロン忘れ・インデント忘れのエラーを直せる
- **前提知識**: ch03-03
- **解説の骨子**
  1. **ifの形** — `if 条件:` + 次行を**半角スペース4つ**下げる。「字下げした行だけが if の支配下」を図解
  2. **インデントは飾りではない** — IndentationError の例。エディタのTabキーで4スペース入ることを案内
  3. **条件が成り立たないと何も起きない** — 値を変えて実行して体感させる
- **実行例サンプルコード**
  ```python
  temperature = 32
  if temperature >= 30:
      print("暑いですね")
  print("ここは必ず表示されます")
  ```
- **演習問題**
  - **問題文の要旨**: 雛形の `temperature = 32` を使い、30以上のときだけ `暑いですね` と表示するif文を書く(判定は32で行うが、値を変えて「何も出ない」ことも試すよう促す)
  - **雛形の方針**: `temperature = 32` のみ与える
  - **判定基準**: ①`_lines == ["暑いですね"]` ②`temperature == 32`
  - **不合格になるべき誤答**: 空回答/条件の向き逆 `<= 30`(何も出力されない)/コロン・インデント忘れ(実行時エラー)
  - **ヒント1**: `if temperature >= 30:` と書き、次の行を4つ字下げして print します。行末の `:` を忘れずに
  - **ヒント2**: `if temperature >= 30:` ⏎ `    print("暑いですね")`
  - **模範解答の方針**: if文2行

### ch04-02 if / else — そうでなければ

- **学習目標**
  1. `else:` で「条件が成り立たないとき」の処理を書ける
  2. `%` と組み合わせて偶数・奇数を判定できる
- **前提知識**: ch04-01, ch03-02
- **解説の骨子**
  1. **elseの形** — ifと同じ高さに `else:`。どちらかが必ず実行される(分かれ道の図)
  2. **偶数・奇数の判定** — `number % 2 == 0`(ch03-02の復習)。定番イディオムとして紹介
- **実行例サンプルコード**: number を偶数/奇数で変えて動かす if/else
- **演習問題**
  - **問題文の要旨**: 雛形の `number = 7` が偶数なら `偶数です`、奇数なら `奇数です` と表示する
  - **雛形の方針**: `number = 7` のみ与える
  - **判定基準**: ①`_lines == ["奇数です"]` ②`number == 7`
  - **不合格になるべき誤答**: 空回答/条件を `% 2 == 1` にしてif/else逆(出力「偶数です」)…は正答同値なので不合格にならない(問題なし)。真の典型誤答は `% 2 = 0`(SyntaxError)や両方表示(行数超過)
  - **ヒント1**: 偶数は「2で割った余りが0」。`if number % 2 == 0:` で分岐し、`else:` に奇数の処理を書きます
  - **ヒント2**: if/else 4行の形をほぼ提示
  - **模範解答の方針**: if/else 4行

### ch04-03 elif — 3つ以上に分ける

- **学習目標**
  1. `elif` で3分岐以上を書ける
  2. 「上から順に判定され、最初に成り立った所だけ実行」を理解する
- **前提知識**: ch04-02
- **解説の骨子**
  1. **elifの形** — if / elif / else の3点セット。elifはいくつでも並べられる
  2. **順番が大事**(本レッスンの核心)— 「90点以上」より先に「70点以上」を書くと、95点でも「よくできました」になってしまう誤り例を実際に見せる
- **実行例サンプルコード**: 正しい順の3分岐(90以上/70以上/それ未満)と、コメントで誤り順の説明
- **演習問題**
  - **問題文の要旨**: 雛形の `score = 95` に対し、90以上なら `すばらしい`、70以上90未満なら `よくできました`、70未満なら `がんばろう` を表示する3分岐を書く
  - **雛形の方針**: `score = 95` のみ与える
  - **判定基準**: ①`_lines == ["すばらしい"]` ②`score == 95`(※scoreが95なので、elifの順序を誤ると「よくできました」が出て不合格になる — 順序ミスを検出する設計)
  - **不合格になるべき誤答**: 空回答/判定順序の誤り(70以上を先に書く)/elifを使わず独立したif3つで重複出力(行数超過)
  - **ヒント1**: 大きい条件から順に書きます。`if score >= 90:` → `elif score >= 70:` → `else:`
  - **ヒント2**: 6行の形をほぼ提示
  - **模範解答の方針**: if/elif/else 6行

### ch04-04 条件の組み合わせとネスト

  2. bool変数をそのまま条件に書ける(`== True` は不要)
- **前提知識**: ch04-03, ch03-04
- **解説の骨子**

  2. **ifの中にif(ネスト)** — 同じ判定をネストで書いた対比例。インデントが深くなるので、andで書ける時はandを推奨
- **実行例サンプルコード**: and版とネスト版の入場判定を並べる
- **演習問題**
  - **問題文の要旨**: 雛形の `age = 20`、`has_ticket = True` を使い、「18歳以上 かつ チケットあり」なら `入場できます`、そうでなければ `入場できません` と表示する
  - **雛形の方針**: 2変数を与える
  - **判定基準**: ①`_lines == ["入場できます"]` ②入力2変数の不変
  - **不合格になるべき誤答**: 空回答/`or` 誤用(この入力では検出不能だが、ヒントで「両方必要ならand」と明記。※検出のため雛形を `age = 20, has_ticket = False` にすると期待出力が「入場できません」になり、or誤用(True)を検出できる — **こちらを採用**。期待: `_lines == ["入場できません"]`)

  - **ヒント1**: 2つの条件が「両方」必要なときは `and`。チケットを持っていない人は入れるでしょうか?
  - **ヒント2**: `if (age >= 18) and has_ticket:` / else で「入場できません」

### ch05-04 while — 条件が成り立つ間くり返す

- **学習目標**
  1. `while 条件:` のループを書ける(カウンタ変数の更新を忘れない)
  2. 無限ループになっても10秒で自動停止することを知り、恐れず試せる
- **前提知識**: ch05-03
- **解説の骨子**
  1. **whileの形** — 「回数が決まっていない繰り返し」に使う。`count = 1` → `while count <= 5:` → 中で `count += 1`
  2. **3点セット** — ①開始前の準備 ②続ける条件 ③中での更新。**③を忘れると無限ループ**
  3. **無限ループは怖くない** — このサイトでは10秒で自動停止する仕組みがあること、実行が止まらないときは停止を待てばよいことを明記(安心設計)
  4. **forとの使い分け** — 回数が決まっているならfor、条件次第ならwhile
- **実行例サンプルコード**
  ```python
  count = 1
  while count <= 5:
      print(count)
      count += 1
  ```
- **演習問題**
  - **問題文の要旨**: whileを使って1から5までを1行ずつ表示する(forは使わない。カウンタの更新を忘れずに)
  - **雛形の方針**: `count = 1` を与え、while以降を書かせる
  - **判定基準**: ①`_lines == ["1", "2", "3", "4", "5"]` ②`count == 6`(ループ終了後の値。更新をループ内で行った証跡として緩く確認 — printを5行直書きした解を弾く副次効果もある)
  - **不合格になるべき誤答**: 空回答/`count += 1` 忘れ(タイムアウト→実行時不合格)/条件 `< 5`(4まで)/print直書き(②で不合格)
  - **ヒント1**: `while count <= 5:` の中で「表示」と「countを1増やす」の2つを行います。増やし忘れると無限ループ(10秒で止まります)
  - **ヒント2**: 4行のコードをほぼ提示
  - **模範解答の方針**: 実行例と同型の4行

### ch05-05 breakとcontinue — ループの流れを変える

- **学習目標**
  1. `break` でループを途中で抜けられる
  2. `continue` でその回だけスキップできる
- **前提知識**: ch05-04
- **解説の骨子**
  1. **break: 打ち切る** — 「5が見つかったら終了」の例。whileの `while True:` + break は紹介のみ(深入りしない)
  2. **continue: 今回だけ飛ばす** — 3の倍数をスキップして表示する例
  3. **ifと組み合わせて使う** — break/continueは通常ifの中に書く。breakを書く位置(printの前か後か)で結果が変わることを見せる
- **実行例サンプルコード**: 1〜10のループでiが4の倍数のときcontinue、8でbreakする例(出力: 1,2,3,5,6,7)
- **演習問題**
  - **問題文の要旨**: 1から10までのforループで、5になったらbreakでループを終える。出力は 1〜4 の4行になるようにする(5は表示しない)
  - **雛形の方針**: `for i in range(1, 11):` と `print(i)` を与え、if+breakを追加させる
  - **判定基準**: `_lines == ["1", "2", "3", "4"]`
  - **不合格になるべき誤答**: 雛形のまま(1〜10)/breakをprintの後に置く(5まで表示される)/条件 `i == 4`(3まで)
  - **ヒント1**: `if i == 5:` のときに `break` します。printの**前**で止めないと5が表示されてしまいます
  - **ヒント2**: `if i == 5:` ⏎ `        break` を printの前の行に入れる
  - **模範解答の方針**: for+if+break 4行

### ch08-01 関数を作る — 処理に名前を付ける

### ch09-02 try / except — エラーに備える

- **学習目標**
  1. `try`/`except` でエラー発生時の代替処理を書ける
  2. 例外の型(ValueError等)を指定してexceptできる
- **前提知識**: ch09-01, ch08-03(関数・return・None)
- **解説の骨子**
  1. **エラーで止めたくない時がある** — 変換できない文字列を int() する場面(ch02-05の伏線回収)
  2. **try/exceptの形** — 「tryの中を実行 → エラーが起きたらexceptに飛ぶ」フロー図
  3. **型を指定する** — `except ValueError:`。何でもキャッチ(裸のexcept)は悪い癖になるので型指定を標準として教える
  4. **関数と組み合わせる** — 「変換できたら数値、できなければNoneを返す」パターン(実務でも頻出の形と紹介)
- **実行例サンプルコード**: `int()` をtry/exceptで包み、成功と失敗の両方を試すコード
- **演習問題**
  - **問題文の要旨**: 文字列を受け取り、`int()` で変換できれば整数を返し、できなければ(ValueError)`None` を返す関数 `to_number(text)` を定義する
  - **雛形の方針**: `def to_number(text):` と `try:` の行まで与える
  - **判定基準**: ①callable ②`to_number("123") == 123` ③`to_number("-5") == -5` ④`to_number("abc") is None`(tryなしだとテスト実行中にValueErrorが伝播して不合格) ⑤`isinstance(to_number("7"), int)`
  - **不合格になるべき誤答**: try無し(④で例外)/文字列のままreturn(⑤)/失敗時に0を返す(④)
  - **ヒント1**: tryの中で `return int(text)`。`except ValueError:` の中で `return None`

### ch10-03 簡易あみだくじ

- **学習目標**
  1. 「あみだくじ=隣どうしの入れ替えの繰り返し」というモデル化を理解する
  2. 引数のリストを壊さないコピー(`list()`)の重要性を知る
- **前提知識**: ch06-02(要素の書き換え), ch06-05(入れ替え), ch05-01, ch08-03
- **解説の骨子**
  1. **あみだくじをデータにする** — 縦線に0番から人が並び、横線は「i番とi+1番の入れ替え」。横線を上から並べたリスト `lines = [0, 1]` で1つのあみだくじが表せる(図解必須: 縦線・横線のASCII図)
  2. **入れ替えは1行** — `positions[i], positions[i + 1] = positions[i + 1], positions[i]`(ch06-05の多重代入の応用)
  3. **引数を壊さない** — 引数のリストを直接並べ替えると呼び出し元のリストまで変わってしまう。`positions = list(players)` でコピーしてから操作する(**本レッスンで新規に解説**)
  4. **コラム: 本物のくじにするには** — randomモジュールの存在だけ紹介(コードは載せるが実行例・演習では使わない)
- **実行例サンプルコード**: `["A", "B", "C"]` と `lines = [0, 1]` で1本ずつ入れ替えて途中経過をprintするコード(演習の答えに近すぎない、コピーなし・関数なしの平置き版)
- **演習問題**
  - **問題文の要旨**: 参加者リスト `players` と横線リスト `lines`(各要素は左側の縦線番号 i で、i番と i+1番を入れ替える)を受け取り、横線を先頭から順に適用した**最終的な並びの新しいリスト**を返す関数 `amida(players, lines)` を定義する。元の `players` を変更してはいけない
  - **雛形の方針**: `def amida(players, lines):` と `positions = list(players)  # コピー` を与える
  - **判定基準**:
    - ①callable
    - ②`amida(["A", "B", "C"], [0, 1]) == ["B", "C", "A"]`(A,B,C → 0で B,A,C → 1で B,C,A)
    - ③`amida(["犬", "猫"], []) == ["犬", "猫"]`(横線なし)
    - ④`amida(["a", "b", "c", "d"], [2, 0, 1]) == ["b", "d", "a", "c"]`
    - ⑤入力の不変性: `_p = ["A", "B", "C"]` を渡して呼んだ後 `_p == ["A", "B", "C"]` であること(コピーし忘れ検出)
    - ⑥戻り値が新しいリストであること: `amida(_p, []) is not _p`
  - **不合格になるべき誤答**: コピーせず直接並べ替え(⑤)/`return players`(⑥)/入れ替えでなく片方向コピー(②)/linesを逆順に適用(④で検出: [2,0,1]は順序依存)
  - **ヒント1**: forで lines から i を1つずつ取り出し、`positions` の i番と i+1番を多重代入で入れ替えます。最後に positions を return
  - **ヒント2**: ループ2行+return の骨格をほぼ提示
  - **模範解答の方針**: 4行の関数+ASCII図と対応させた使用例
